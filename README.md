

이 프로젝트는 하드웨어 제어와 서비스 로직을 분리한 계층형 구조로 설계되었습니다.




[ 계층별 상세 설명 ]


1. Drv (Driver Layer)

- 역할: 신호의 입력 및 전기 발생. MCU 레지스터 직접 제어.

- 수정 시점: MCU 모델 변경이나 센서/부품 등 하드웨어가 바뀔 때 수정합니다.



2. Bsp (Board Support Package Layer)

- 역할: 신호의 해석 및 의미 부여. 

- 수정 시점: 하드웨어 구성 요소는 같으나 연결 핀이 바뀌는 등 회로도가 변경될 때 수정합니다.



3. Task (Scheduling Layer)

- 역할: 신호의 수행 및 시간 관리. CPU 실행 권한을 배분합니다.

- 수정 시점: 시스템 응답성 개선이나 작업 우선순위를 조절해야 할 때 수정합니다.



4. App (Application Layer)

- 역할: 신호의 연산 및 판단. 기획된 시나리오 로직을 실행합니다.

- 수정 시점: 제품의 기능 기획이나 작동 시나리오가 변경될 때 수정합니다.



5. Common (Library \& Protocol)

- 역할: 데이터 규격집. 함수/변수 선언 및 통신 프로토콜 정의.

- 설명: 모든 계층에서 공통적으로 사용하는 약속을 정의하는 부분입니다.




[ 헤더 참조 및 캡슐화 ]


-Common Header: `Drv`, `Bsp`, `Task` 계층은 공통 규격 확보를 위해 `Common` 헤더를 삽입하여 사용합니다.

- App Header: `App` 계층은 독립적인 헤더를 생성합니다. 이는 특정 기능의 캡슐화를 통해 불필요한 정보 노출을 막고 코드의 독립성을 유지하기 위함입니다.
